StrelokC Blog - Lines & Mouse input

Mouse input achieved.

The Details

Well hello, welcome to my blog. If you haven't read the first one then you probably should, it will give some background and context. Or don't, I can't force ya.
Anyway, as you can see I've managed to implement mouse input and feedback in my Graphics Project. It was relatively simple for the most part, nothing difficult, just time consuming. To get it to work I had to set up whats called a callback function and implement a static class. Not 100% sure if 'static class' is the correct term, but its basically a class that has all static members.

Now in order to correctly place the points on the screen for rendering, I had to convert the pixel coordinates to NDC (Normalised Device coordinates). Additionally I had to transform them in pixel-space to that the origin is at the centre of the screen. After that it was a matter of linking together the input class with the service locator class (static class) so that anytime you click in the graphics window, it'd create a point under the cursor.

The issues I ran into were:

  • The program not compiling, which was fixed by defining the static class member variables inside the .cpp file.
  • Mouse input not doing anything. This was caused by passing in the wrong variable to glBufferData(). Specifically the size of the data being sent to the GPU. I was sending sizeof(std::vector<>) instead of pointCount * sizeof(float) * 3.
